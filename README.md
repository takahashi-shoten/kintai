# 高橋商店 勤怠管理アプリ（最小機能版 v1）

建設業・大工業の従業員勤怠を、スマホの音声入力で毎日かんたんに記録・集計できる
レスポンシブ Web アプリ（PWA）です。React などは使わず、`index.html` 1ファイルで動きます。

- 対応：iPhone Safari / Android Chrome / PC ブラウザ
- データ保存：その端末の localStorage（ブラウザ内）
- 既存の「キャッシュ管理アプリ」（親フォルダの index.html）とは別プロジェクトです

---

## ファイル構成

```
kintai/
├ index.html          … アプリ本体（画面・解析・保存・集計・CSV すべて）
├ manifest.json       … PWA 設定（ホーム画面追加用）
├ service-worker.js   … オフライン対応
├ icon-192.png        … アイコン
└ icon-512.png        … アイコン
```

## 起動方法

### ① ローカルサーバーで起動（推奨：PWA・スマホ実機で使う場合）

Mac 標準の Python で、`kintai` フォルダの中で実行します。

```bash
cd "～/Library/Mobile Documents/com~apple~CloudDocs/takahashi-cash-manager/kintai"
python3 -m http.server 8080
```

ブラウザで `http://localhost:8080/` を開きます。

### ② ファイルを直接開く（動作確認だけならこれでOK）

`index.html` をダブルクリックするとブラウザで開きます。
※ この方法だと PWA（ホーム画面追加・オフライン）は無効になります。

---

## スマホ実機・ホーム画面に追加（PWA）

1. PC とスマホを同じ Wi-Fi につなぐ
2. PC で上記①のサーバーを起動
3. PC の IP アドレスを確認（システム設定 → Wi-Fi → 詳細 → IP アドレス。例 `192.168.1.10`）
4. スマホのブラウザで `http://192.168.1.10:8080/` を開く
   - **iPhone（Safari）**：共有ボタン → 「ホーム画面に追加」
   - **Android（Chrome）**：右上メニュー → 「ホーム画面に追加 / アプリをインストール」
5. 追加されたアイコンから、アプリのように起動できます

## 使い方

### 勤怠を入力する
1. ホームの「🎤 今日の勤怠を入力」をタップ（日付は自動で今日）
2. 音声入力欄をタップし、**キーボードのマイク**で話す
   - 例：「砂川市 明山邸 8時から18時 交通費5000円 遠方経費2000円」
   - 例：「札幌 佐藤邸 8時 17時半 交通費2000円」
   - 例：「東6条 現場 休み」
3. 「この文章を解析して下に反映」をタップ → 現場名・時間・交通費などに自動振り分け
4. 必要なら手で直して「保存する」

### 解析が理解できる言い方
- 時間：`8時` `18時` `17時半` `8時30分` `8:00` `午後5時半`（最初が開始、次が終了）
- 交通費：`交通費5000円` `交通費0円`
- 遠方経費：`遠方経費2000円` `遠方0円`
- 休憩：`休憩1時間` `休憩45分`（省略時は60分）
- 区分：`休み` `半日`（無ければ「出勤」）
- 残った言葉が現場名になります

### 確認・集計
- **月別一覧**：その月の記録を一覧。タップで修正・削除
- **月別集計**：出勤日数 / 実働時間 / 交通費 / 遠方経費の合計と、現場別一覧
- **社長確認画面**：全記録の確認と明細表、**CSV出力**（Excel で開けます）

---

## データについて
- 記録はその端末のブラウザ内（localStorage）に保存されます
- 別の端末とは共有されません（次の拡張で対応予定）
- ブラウザのデータ消去をすると記録も消えます。月末は CSV 出力で控えを残すと安心です

## 今後の拡張（設計済み）
このアプリは後から機能追加しやすい構造にしています。

- **データ共有（Firebase / Firestore）**：`index.html` 内の `Store` オブジェクト
  （`getAll` / `getByMonth` / `save` / `remove` など）が保存処理を一手に担っており、
  メソッドはすべて async（Promise）です。中身を Firestore 呼び出しに差し替えるだけで、
  画面側のコードを変えずにクラウド共有へ移行できます。
- **従業員ごとのログイン**：保存データに `employeeName` を持たせ、社長確認画面は
  従業員ごとにグループ集計しています。複数人＋認証への拡張を見込んだ作りです。
- **給与計算**：実働時間・各種合計をすでに集計しているため、単価設定などを足して拡張できます。
